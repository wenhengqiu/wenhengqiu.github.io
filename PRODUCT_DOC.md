```
┌─────────────────────────────────────────────────────────────┐
│                    Info-Getter v1.0                         │
├─────────────────────────────────────────────────────────────┤
│  ┌──────────┐  ┌──────────┐  ┌──────────┐  ┌──────────┐   │
│  │ Fetcher  │→ │Translator│→ │ Publisher│→ │ Scheduler│   │
│  │  采集    │  │  翻译    │  │  发布    │  │  调度    │   │
│  └──────────┘  └──────────┘  └──────────┘  └──────────┘   │
│        ↓             ↓             ↓             ↓         │
│   RSS/API      OpenClaw/Kimi   GitHub Pages   每小时执行   │
│   12个源       自动翻译         自动部署       守护进程     │
└─────────────────────────────────────────────────────────────┘
```

### 采集流程

```
每小时自动执行：
1. 扫描 12 个信息源 (RSS/API)
2. 采集新文章 (标题、摘要、URL)
3. OpenClaw/Kimi API 翻译 (中文标题和摘要)
4. 质量评分 (0-1 分)
5. 分类打标 (llm/autonomous/robotics/zhuoyu)
6. 去重检测 (基于 URL)
7. 保存到 JSON 文件
8. 推送到 GitHub
9. GitHub Pages 自动部署
```

## 每日信息总结功能 (新增)

### 功能描述
每次 Info-Getter 采集新文章时，自动生成今日要点总结：

1. **提取核心事件**
   - 识别最重要的 3-5 条新闻
   - 按重要性排序

2. **生成要点摘要**
   - 一句话概括每条新闻
   - 提取关键数据和信息

3. **分类统计**
   - 今日新增文章数
   - 各分类占比

4. **展示位置**
   - 首页顶部 "今日要点" 区域
   - 状态页面 "今日更新" 统计

### 实现方式

**方案 1：AI 自动生成 (推荐)**
```python
# 采集完成后，调用 OpenClaw/Kimi API
def generate_daily_summary(articles):
    prompt = f"""
    基于以下今日采集的 {len(articles)} 篇文章，生成今日要点总结：
    
    文章列表：
    {articles_text}
    
    请输出：
    1. 3-5 条核心新闻要点（一句话概括）
    2. 今日总体趋势总结（100字以内）
    3. 各分类统计
    """
    
    summary = call_kimi_api(prompt)
    return summary
```

**方案 2：规则提取**
- 按 quality_score 排序取 Top 5
- 提取标题生成要点
- 统计各分类数量

## 使用说明

### 对于用户

**查看今日资讯：**
1. 访问 https://wenhengqiu.github.io/
2. 阅读 "今日要点" 了解核心内容
3. 浏览文章列表，点击感兴趣的标题
4. 点击 "阅读原文" 查看完整内容
5. 使用分类按钮筛选特定领域

1. 访问 https://wenhengqiu.github.io/status.html
2. 查看守护进程运行状态
