# Info-Getter Web GUI 产品需求文档

## 1. 产品概述

### 1.1 产品名称
**Info-Getter Dashboard** - Info-Getter可视化管理平台

### 1.2 产品定位
为Info-Getter提供可视化管理界面，方便监控、管理和调试自动采集系统。

### 1.3 核心价值
- **可视化管理**: 无需命令行，图形界面操作
- **实时监控**: 查看采集状态、日志、文章列表
- **灵活控制**: 手动开关、调整规则、即时生效
- **数据分析**: 统计报表、趋势分析

## 3. 技术架构

### 3.1 前端
- **框架**: React + TypeScript
- **UI库**: Ant Design
- **图表**: ECharts
- **状态管理**: Redux Toolkit

### 3.2 后端
- **框架**: Flask (Python)
- **API**: RESTful + WebSocket
- **数据库**: SQLite (配置) + JSON (文章)
- **认证**: JWT Token

### 3.3 部署
- **方式**: 与Info-Getter一起部署
- **访问**: http://localhost:8080
- **静态文件**: 打包到GitHub Pages

## 5. API设计

### 5.1 文章API
```
GET    /api/articles              # 获取文章列表
GET    /api/articles/:id          # 获取单篇文章
PUT    /api/articles/:id          # 更新文章
DELETE /api/articles/:id          # 删除文章
POST   /api/articles/batch-delete # 批量删除
```

### 5.2 信息源API
```
GET    /api/sources               # 获取源列表
POST   /api/sources               # 添加源
PUT    /api/sources/:id           # 更新源
DELETE /api/sources/:id           # 删除源
POST   /api/sources/:id/test      # 测试源
```

### 5.3 系统API
```
GET    /api/stats                 # 获取统计数据
GET    /api/logs                  # 获取日志
GET    /api/status                # 获取系统状态
POST   /api/collect               # 手动触发采集
POST   /api/control/start         # 启动守护进程
POST   /api/control/stop          # 停止守护进程
```

### 5.4 WebSocket
```
ws://localhost:8080/ws/logs       # 实时日志
```
